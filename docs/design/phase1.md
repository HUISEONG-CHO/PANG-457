# Phase 1 설계 — 메인 화면 UI

## 목표

브라우저 실행 시 가장 먼저 보이는 메인 화면을 구현한다.  
이 단계에서는 게임 로직 없이 **UI와 버튼 동작**만 완성한다.

---

## 화면 레이아웃

```
┌──────────────────────────────────────┐
│                                      │
│                                      │
│               PANG                   │  ← 타이틀 텍스트
│                                      │
│                                      │
│           [ 게임 시작 ]               │  ← 버튼
│           [ 게임 종료 ]               │  ← 버튼
│                                      │
│                                      │
└──────────────────────────────────────┘
```

- 전체 화면 중앙 정렬
- 타이틀 → 버튼 순서로 세로 배치
- 배경: 단색 (검정 계열, 아케이드 분위기)

---

## 컴포넌트 구조

```
App.tsx
└── MainScreen.tsx
      ├── 타이틀 영역
      └── 버튼 영역
            ├── 게임 시작 버튼
            └── 게임 종료 버튼
```

### App.tsx
- `screen` 상태로 현재 화면을 관리 (`'main' | 'game'`)
- Phase 1에서는 `'main'` 화면만 렌더링
- 이후 Phase에서 `'game'` 화면이 추가될 예정

```ts
type Screen = 'main' | 'game'
```

### MainScreen.tsx
- `onStart` 콜백: 게임 시작 버튼 클릭 시 App의 screen 상태를 `'game'`으로 변경 (Phase 2에서 연결)
- Phase 1에서는 클릭 시 `console.log` 또는 `alert`로 동작 확인
- `onExit` 콜백: 게임 종료 버튼 클릭 시 종료 확인 다이얼로그 표시

---

## 버튼 동작 상세

### 게임 시작
- 클릭 시 → Phase 1: `alert('게임 시작!')` 출력 (Phase 2에서 실제 화면 전환으로 교체)

### 게임 종료
- 클릭 시 → `confirm('게임을 종료하시겠습니까?')` 다이얼로그 표시
- 확인 → `window.close()` 시도
  - 브라우저 정책상 직접 열린 탭이 아니면 닫히지 않을 수 있음 → 안내 메시지로 대체
- 취소 → 메인 화면 유지

## 파일 구성

```
src/
├── App.tsx                   ← screen 상태 추가
├── App.css                   ← 기존 스타일 초기화
├── screens/
│   ├── MainScreen.tsx        ← 신규 생성
│   └── MainScreen.css        ← 신규 생성
```

---

## 스타일 방향

| 항목 | 내용 |
|------|------|
| 배경색 | `#000` (검정) |
| 타이틀 폰트 | 굵고 큰 폰트, 흰색 또는 노란색 계열 |
| 버튼 스타일 | 테두리(border) 스타일, hover 시 색 반전 |
| 레이아웃 | `flexbox` 세로 중앙 정렬, 전체 화면(`100vw x 100vh`) |

---

## 미결 사항 (검토 요청)

1. **타이틀 폰트** — 픽셀 폰트(레트로 스타일) 적용 여부
2. **게임 종료 동작** — `window.close()` 불가 시 단순 안내 메시지로 대체할지
3. **배경** — 단색 외에 배경 이미지나 애니메이션 적용 여부 (Phase 11로 미룰지)
