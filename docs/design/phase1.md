# Phase 1 설계 — 메인 화면 UI

## 목표

브라우저 실행 시 가장 먼저 보이는 메인 화면을 구현한다.  
이 단계에서는 게임 로직 없이 **UI, 버튼 동작, 키보드 메뉴 탐색**만 완성한다.

---

## 화면 레이아웃

```
┌──────────────────────────────────────┐
│                                      │
│               PANG                   │  ← 타이틀 텍스트
│                                      │
│           ▶ 게임 시작                 │  ← 선택된 항목 (포커스)
│             게임 종료                 │
│                                      │
└──────────────────────────────────────┘
```

- 전체 화면 중앙 정렬
- 타이틀 → 메뉴 순서로 세로 배치
- 배경: 단색 (검정 계열, 아케이드 분위기)

---

## 컴포넌트 구조

```
App.tsx
└── MainScreen.tsx
      ├── 타이틀 영역
      └── 메뉴 영역
            ├── 게임 시작 항목
            └── 게임 종료 항목
```

### App.tsx
- `screen` 상태로 현재 화면을 관리 (`'main' | 'game'`)
- Phase 1에서는 `'main'` 화면만 렌더링

```ts
type Screen = 'main' | 'game'
```

### MainScreen.tsx
- `selectedIndex` 상태로 현재 포커스된 메뉴 항목 관리 (기본값: `0`)
- `useEffect`로 `keydown` 이벤트 등록/해제
- `onStart` 콜백: Phase 1에서는 `alert('게임 시작!')` 출력
- `onExit` 콜백: `confirm` 다이얼로그 → `window.close()` 시도

---

## 키보드 조작

| 키 | 동작 |
|----|------|
| `↑` | 이전 항목으로 이동 (최상단에서 최하단으로 순환) |
| `↓` | 다음 항목으로 이동 (최하단에서 최상단으로 순환) |
| `Enter` | 현재 선택된 항목 실행 |
| 마우스 클릭 | 해당 항목 직접 실행 |

### 순환 로직
```ts
// 다음 항목 (↓)
setSelectedIndex((prev) => (prev + 1) % MENU_COUNT)

// 이전 항목 (↑)
setSelectedIndex((prev) => (prev - 1 + MENU_COUNT) % MENU_COUNT)
```

---

## 포커스(선택) 표시

- 선택된 항목 앞에 `▶` 마커 표시
- 선택된 항목: 노란색(`#ffe600`) 텍스트
- 비선택 항목: 회색(`#888`) 텍스트

---

## 버튼 동작 상세

### 게임 시작 (index: 0)
- 실행 시 → `alert('게임 시작!')` (Phase 2에서 실제 화면 전환으로 교체)

### 게임 종료 (index: 1)
- 실행 시 → `confirm('게임을 종료하시겠습니까?')` 다이얼로그
- 확인 → `window.close()` 시도 (불가 시 안내 메시지)
- 취소 → 메인 화면 유지

---

## 파일 구성

```
src/
├── App.tsx                   ← screen 상태 관리
├── App.css                   ← 초기화
├── screens/
│   ├── MainScreen.tsx        ← selectedIndex 상태 + keydown 이벤트
│   └── MainScreen.css        ← 선택/비선택 스타일
```

---

## 스타일 방향

| 항목 | 내용 |
|------|------|
| 배경색 | `#000` (검정) |
| 타이틀 폰트 | 굵고 큰 폰트, 노란색(`#ffe600`) |
| 선택 항목 | 노란색 텍스트 + `▶` 마커 |
| 비선택 항목 | 회색(`#888`) 텍스트 |
| 레이아웃 | `flexbox` 세로 중앙 정렬, 전체 화면(`100vw x 100vh`) |

---

## 미결 사항 (검토 요청)

1. **타이틀 폰트** — 픽셀 폰트(레트로 스타일) 적용 여부
2. **게임 종료 동작** — `window.close()` 불가 시 단순 안내 메시지로 대체할지
3. **배경** — 단색 외에 배경 이미지나 애니메이션 적용 여부 (Phase 11로 미룰지)
